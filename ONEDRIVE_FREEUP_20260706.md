## 4. ⚠️ 회수 정지 원인 진단

- 로컬 잔여 96,564 MB가 6분간 **0 감소**, OneDrive.exe CPU **952.4에서 고정**(유휴, WS 121MB) → 디하이드레이션 능동 정지.
- 잔여의 대부분이 **소수 초대형 파일에 집중**:
  - **`4_보관\법인계약.zip` = 43,933 MB(≈43.9 GB) 단일 파일** — Unpinned=True 이나 Offline=False(클라우드 미확정).
  - 2_상시업무 잔여 50,622 MB도 대형 파일 7,710개에 분산.
- **판정:** OneDrive는 **클라우드 업로드가 확정되지 않은 파일의 로컬 사본을 비우지 않음**(데이터 보호 설계). C: 임계 부족 상태에서 대형 파일 업로드가 미완/정체된 것으로 보이며, 이는 `attrib`로 해결 불가한 **외부(동기화/업로드/쿼터) 조건**.
- 참고: OneDrive 계정 2종(`Business1` / `Personal`) 존재. 대상 폴더 소속 계정의 업로드/쿼터 상태 점검 필요.

### 권고(사용자 결정 필요 · 본 작업 범위 밖)
1. OneDrive 동기화 재개/강제 업로드 후, 대형 파일 업로드 완료를 기다렸다가 재측정하면 나머지 ~94 GB 자동 회수 가능.
2. `법인계약.zip`(43.9GB)은 개별 업로드 확인 또는 외장/별도 보관 검토.
3. 업로드 완료 후 동일 `attrib +U` 재적용 없이도 디하이드레이션이 이어짐(언핀 이미 설정됨).
