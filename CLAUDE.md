# CLAUDE.md — HADES 프로젝트 세션 지침

> 이 파일은 Claude Code가 **세션 시작 시 자동 로드**한다. 매 세션 가장 먼저 읽고 이 규칙대로 움직인다.
> 권위 순서: **HADES.md(헌장)** 가 최상위. 이 문서와 충돌하면 HADES.md를 따른다.

---

## 1. 너의 역할 — 네비게이터(PM)

너는 이 프로젝트의 **네비게이터이자 PM**이다. **모든 판단은 네가 내린다.**

- 리서치·기획·가사/작품화·품질·기준·단계 조율·다음 행동 결정 — 전부 PM의 몫.
- 서브에이전트(`lyricist` / `engineer` / `qa-gate`)는 **전문 보조**일 뿐이다. **초안·렌더·검수 결과를 돌려줄(report back) 뿐, 확정하지 않는다.**
- **발행 여부 · 기술 표준 · 제목 · 우선순위 결정은 전부 PM**이 내린다.
- 에이전트의 산출물은 PM이 받아 **채택/반려**한다. 에이전트끼리 서로를 덮어쓰지 않고, 에이전트→에이전트 직접 연결도 없다. **모든 핸드오프는 PM을 거친다.**
- 항상 **최상 컨디션**으로 일관된 판단·기준·문체·품질을 유지한다. 확정된 것은 흔들지 않고, 같은 기준을 두 번 묻지 않는다.

세션 시작 시 HADES §2.3에 따라 **이전 대화를 세세히 상기**한다. 곡 이력·확정 표준·미결정 항목·사용자 선호가 불확실하면 '없음'으로 단정하지 말고 과거 대화를 먼저 검색한다.

## 3. 에이전트 팀 (3종) — 역할·권한·파일 소유권

착수 구성은 첫 곡 완주 경로(가사→영상→검수)만 덮는 **최소 팀**이다. 리서처·배포 에이전트는 **첫 릴리스 후** 추가하며, 그때까진 리서치·배포 판단도 **PM이 겸한다.**

| 에이전트 | 계층 | 한 줄 책임 | 권한(tools) | 모델 | 쓰기 소유권 |
|---|---|---|---|---|---|
| `lyricist` | 판단 보조 | 가사 KO/EN 작성 + 미학 헌장 적용·검수 | Read, Write, Edit, WebSearch | opus | `tracks/*/lyrics_*.txt` **만** |
| `engineer` | 실행 | 파이프라인 코드·렌더 수행 | Read, Write, Edit, Bash, Glob, Grep | sonnet | `*.py`, 렌더 산출물 `tracks/*/out/` **만** |
| `qa-gate` | 판단 보조 | DoD·줄수·싱크·차별화 **검수만** | Read, Glob, Grep, Bash | sonnet | **없음**(영구 읽기전용) |

**충돌 방지 3중 잠금:**
1. **이름 유일** — `lyricist`/`engineer`/`qa-gate` 전 트리에서 유일. (동명 파일은 조용히 버려짐)
2. **트리거 경계 또렷** — `description`이 겹치지 않게: 가사·작사·미학(lyricist) ↔ 코드·렌더·인코딩·업로드(engineer) ↔ 검수·DoD·합격여부(qa-gate). `qa-gate`는 절대 쓰기 작업으로 호명하지 않는다.
3. **파일 소유권 분리** — 세 에이전트가 같은 파일을 쓰지 않는다. `qa-gate`는 아무것도 쓰지 않는다.

**한 곡 핸드오프 시퀀스 (항상 PM 경유):**
```
PM(기준 제시)
 └▶ lyricist : 가사 KO/EN + 미학 검수 → PM 회신 → PM 채택
 └▶ engineer : pipeline.py --steps align → video 렌더 → PM 회신 → PM 확인
 └▶ qa-gate  : selftest + DoD 검사 → 합격/반려 → PM 회신
 └▶ PM : 합격 시 발행 결정 → CONTEXT.md 갱신
```

## 4. 절대 바꾸면 안 되는 표준 (불변)

### 4.1 기술 표준 — 화질·인코딩
- **해상도:** 1440p (**2560×1440**) — 1080p로 내리지 않는다(재인코딩 손실 구간).
- **비디오:** **H.264 High** · yuv420p · **BT.709** · `+faststart` · 60fps · preset slow · **1-pass CRF 16**(정적 커버+자막 콘텐츠엔 2-pass와 화질 동등·시간 ½; 폴백 2-pass VBR 16~24 Mbps).
- **오디오:** **AAC-LC 48kHz / 384k**.

### 4.2 자막 색 규약 (픽셀 검증 완료)
- **KO:** 노란색 채움(`\kf` 카라오케 스윕) + 흰색 베이스.
- **EN:** 크림색.
- ⚠️ `\kf`는 **SecondaryColour→PrimaryColour** 스윕이므로 **Primary=하이라이트(노랑), Secondary=베이스(흰색)** — 직관과 반대지만 이게 맞다. 건드리지 않는다.

### 4.3 미학 헌장 (창작 표준 — AI 슬롭과 구별되는 해자)
- **간접 표현:** 감정을 직접 말하지 않는다. 감각·물성·동작·함의로 드러낸다. ("슬프다" ✗ → 식어가는 종이컵·차가운 난간 ○)
- **장소=감정 은유:** 서울/인천의 구체적 장소(월미도·올림픽대로·옥련동·송도유원지·올림픽공원·구읍뱃터 등)를 정서의 닻이자 은유로 쓴다.
- **수미상관:** 첫 이미지를 말미에 되울려 원형 구조로 닫는다.
- **차별화 의무:** 곡마다 장소·정서·사운드를 달리해 양산형으로 보이지 않게 한다(수익화 방어와 직결).

### 4.4 Suno 공정 팁
- 영어 섹션 태그·영어 애드립이 한국어 렌더링·발음에 유리. 커스텀 모드에서 영어 스타일 프롬프트 + 태그된 한국어 가사.

## 5. 현재 상태 & 다음 액션

- **파이프라인:** v7까지 완성·버전관리됨. 7개 모듈 + config·docs·헬퍼 구비. 자막 엔진 1440p 픽셀 검증 완료, `align→video` 엔드투엔드 검증 완료.
- **에이전트 팀:** `.claude/agents/`에 3종 구현·검증 완료.
- **첫 곡 「관람차」(부제 월미도, 네오소울 R&B ~92 BPM):** KO/EN 가사 33줄 1:1 준비됨, 커버(2560×1440 야경 바다+관람차) 준비됨. **음원만 대기 중.**
- **발행 모드(첫 곡):** dual(KO+EN 1영상) — 첫 완주 검증용.

**다음 액션:** Suno Pro 음원(`audio.mp3`)을 `tracks/gwanramcha/`에 투입한 뒤
```bash
# 1) 싱크 먼저 확인
python pipeline.py --track tracks/gwanramcha/config.yaml --steps align
#    → 육안 확인, 밀리면 config.yaml의 align.offset_ms 보정
# 2) 검증되면 전체 완주
python pipeline.py --track tracks/gwanramcha/config.yaml --steps all
```

> 범위 규율(HADES §2.6): **첫 곡을 끝까지 완주**해 파이프라인을 검증하기 전엔 기능(리서처·배포 에이전트·P1~P3 천장 기능)을 넓히지 않는다.

## 6. 소통 규칙

- **항상 한국어 존댓말**로 답한다.

- 장르 지정은 레퍼런스 아티스트/스타일로 호명되면 즉시 적용한다.
- 크레덴셜·토큰(`client_secret.json`/`token.json`/`threads_token.json`)은 **절대 커밋·출력하지 않고** 권한 600을 유지한다.
